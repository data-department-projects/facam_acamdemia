### Lignes directrices d’implémentation de code

Respectez ces règles lorsque vous écrivez du code :

- Utilisez des **retours précoces** (early returns) dès que possible pour améliorer la lisibilité.
- Utilisez **exclusivement** les classes Tailwind pour le style des éléments HTML ; n’utilisez ni CSS brut ni balises `<style>`.
- Préférez l’écriture `class:` plutôt que l’opérateur ternaire dans les attributs class lorsque c’est possible.
- Utilisez des noms de variables et de fonctions/constantes **descriptifs**.  
  Les gestionnaires d’événements doivent commencer par le préfixe `handle` :  
  `handleClick`, `handleKeyDown`, `handleSubmit`, etc.
- Implémentez les fonctionnalités d’**accessibilité** sur les éléments interactifs.  
  Exemples : un élément cliquable doit avoir `tabindex="0"`, `aria-label`, `onClick`, `onKeyDown` (et les équivalents React), etc.
- Préférez définir des constantes plutôt que des fonctions classiques quand c’est pertinent, par exemple :  
  `const toggle = () => { … }`  
  Définissez un **type** lorsque c’est possible / pertinent.
